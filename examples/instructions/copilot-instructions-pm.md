# Project Context
This is an inventory management system for a warehouse operations team.
The API supports product tracking, stock level monitoring, supplier management, and automated reorder alerts.

## Business Context
- The warehouse manages ~500 SKUs across 5 product categories
- Key users: warehouse staff (viewers), inventory managers, and system admins
- Critical business process: reorder alerting when stock drops below threshold
- The system serves both the warehouse floor (mobile) and the management dashboard (web)

## When Helping Me
- Explain technical concepts in business terms
- Reference API endpoints by their purpose, not their code path
- Frame suggestions in terms of user impact and business value
- When discussing features, use the acceptance criteria format: "Given [context], When [action], Then [result]"
- Include effort estimates when suggesting new features (S/M/L/XL)
- Always consider the impact on existing users when suggesting changes

## Communication Preferences
- Use bullet points for lists, not paragraphs
- Keep summaries under 200 words
- Include a "TL;DR" for anything longer than 3 paragraphs
- When listing risks, pair each with a mitigation strategy
