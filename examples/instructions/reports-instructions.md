When generating or editing reports in this project:

## Format Standards
- Start every report with: Title, Report Date, Author, Period
- Include an Executive Summary section (3 sentences max)
- Use markdown tables for metrics and comparisons
- Format dates as YYYY-MM-DD
- Currency values include two decimal places and the $ symbol

## Required Sections
1. Executive Summary
2. Accomplishments (bullet points)
3. In Progress (bullet points with % complete)
4. Risks & Blockers (table with Risk, Impact, Mitigation columns)
5. Metrics (table with current vs. previous period)
6. Next Steps

## Tone & Language
- Professional but concise — target 300-500 words
- Use active voice: "Shipped the reporting endpoint" not "The endpoint was shipped"
- Quantify impact where possible: "Reduced response time by 33%" not "Improved response time"
- When listing risks, always pair with a mitigation strategy
- Avoid jargon — if a technical term is needed, briefly explain it
