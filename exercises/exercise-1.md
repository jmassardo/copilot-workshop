# Exercise 1: Your First Non-Code Task

**Block**: 1 — Copilot for Everyone  
**Time**: 8 minutes  
**Goal**: Use Copilot for a task relevant to YOUR role — not coding

---

## Instructions

Pick **one** task from the table below based on your role. You can do this in **VS Code Chat** or on **GitHub.com** — whichever feels more natural.

### Option A: VS Code Chat

Open Copilot Chat (`Ctrl/Cmd + Shift + I`) and type your prompt.

### Option B: GitHub.com

Navigate to this repo on github.com, click the Copilot icon, and type your prompt.

---

## Role-Specific Tasks

### Project Manager

**VS Code Chat:**
> Create a project status report template with sections for risks, blockers, accomplishments, and next steps

**GitHub.com:**
> Open this repo → Copilot Chat → "What were the most significant changes to this repo in the last month?"

### Product Manager

**VS Code Chat:**
> Draft user acceptance criteria for a feature that lets users export their inventory data as CSV

**GitHub.com:**
> Open an issue → Copilot Chat → "Suggest acceptance criteria for this feature request"

### Tech Writer

**VS Code Chat:**
> Write a getting-started guide for a CLI tool called `inventory-cli` that takes `--env` and `--format` flags

**GitHub.com:**
> Navigate to `src/routes/items.ts` on github.com → Copilot → "Explain this file so I can write documentation for it"

### DBA

**VS Code Chat:**
> Write a PostgreSQL query to find the top 10 largest tables by row count and disk usage in the inventory database

**GitHub.com:**
> Open `sql/queries/common-reports.sql` on github.com → Copilot → "Review these queries for performance issues"

### Manager

**VS Code Chat:**
> Draft a performance review self-assessment prompt covering impact, collaboration, and growth areas

**GitHub.com:**
> Open a PR → Copilot Chat → "Summarize this PR for a non-technical stakeholder"

### Developer

**VS Code Chat:**
> Open `src/middleware/auth.ts`, select the `authenticate` function → Chat → "Explain what this function does and suggest improvements"

**GitHub.com:**
> Open a PR → Copilot Chat → "What are the riskiest changes in this PR?"

---

## Debrief Questions

After completing your task:

1. What did you ask?
2. How useful was the response?
3. What would you change about your prompt to get a better answer?

> **Key insight**: The quality of the output depends on the specificity of your input. This is the foundation for everything in Block 2.
